# CHUCHUTEA 知识库 — 全系统审计报告

## 点1：UI架构 — 当前是硬编码

**现状：**
- 所有样式通过 `const S = {...}` 对象内联定义
- JSX 中大量重复的 `style={{...}}` 内联样式
- 一个 App.tsx 超过 380 行，包含所有逻辑+样式+JSX
- 没有组件拆分

**需要改为：**
- 拆出独立组件：Sidebar.tsx / ChatArea.tsx / Settings.tsx / InputBox.tsx
- 样式移到 CSS 文件（使用 className）
- App.tsx 只保留顶层状态管理和路由

**改完后效果：**
- 代码可维护，改一个组件不影响其他
- CSS 可以用 hover 伪类（如 `.conv-item:hover`）
- 新人也能看懂结构

---

## 点2：全站俄语界面

**现状：**
- 只有 5 处用了 i18n 翻译（模块标签、欢迎语、输入框、城市名、占位符）
- 顶栏、侧边栏按钮、设置面板、登录页、知识管理面板 — 全是硬编码中文
- 切语言不会刷新页面，只改当前对话的 lang 字段

**需要改为：**
- 所有 UI 文字走 `t()` 函数
- 覆盖：登录页、设置面板的所有文字、侧边栏按钮文本、知识管理面板、消息标签（知识库原文/仅供参考/引用）
- 切换语言时页面所有文字立即生效

**改完后效果：**
- 点 🇷🇺 Русский → 整个页面变俄文：按钮、标签、提示、占位符
- 点 🇨🇳 中文 → 全部恢复中文

## 点3：左侧导航（新对话/删除/置顶/改名）

**现状：**
- ✅ 改名 `renameConv` 逻辑存在
- ✅ 删除 `deleteConv` 逻辑存在  
- ✅ 置顶 `togglePin` 逻辑存在
- ❌ 按钮只在 hover 时显示，但 CSS hover 没生效（`.conv-item:hover .conv-actions` 在 React 内联样式中不工作）
- ❌ "双击改名" tooltip 还在，但双击没反应
- ❌ renameConv 调用时没有弹出输入框

**需要改为：**
- 使用 CSS 文件（非内联）让 hover 生效
- 改名：hover → 点 ✎ → 弹出内联编辑输入框 → 回车确认
- 删除：hover → 点 🗑 → confirm → 删除
- 置顶：hover → 点 📌 → 立即生效

**改完后效果：**
- 鼠标悬停对话 → 三个按钮出现
- 点 ✎ → 输入新名 → 回车 → 名字焕新
- 点 📌 → 对话飞到列表最上面
- 点 🗑 → 确认 → 对话消失

## 点4：知识引用/来源标注

**现状：**
- 后端返回 `source` 和 `references` 字段
- 前端显示了 "📄 知识库原文 · 引用：xxx"
- ❌ 引用没有可点击的链接
- ❌ 引用没有高亮/卡片样式
- ❌ LLM 降级时只说 "⚡ 仅供参考"，没有解释为什么

**需要改为：**
- 知识库命中：显示来源卡片（模块名 + 标题 + 可点击复制）
- 显示检索到了几条、用了哪条
- LLM 降级：明确说 "知识库中未找到，以下为 AI 常识回答"

**改完后效果：**
- 每个知识库回复下面有一条引用卡片 📄
- 员工知道信息来源，可溯源

## 点5：文件上传

**现状：**
- 后端 API `/api/import/upload` 正常
- 前端 KnowledgePanel 有 `<input type="file">`
- ❌ 上传触发无限文件选择器弹窗（React 渲染 bug）
- ❌ 上传后 fetch 被 ngrok 拦截

**需要改为：**
- 使用 button + hidden input 模式（不用原生 onChange 触发）
- 上传到 /api/import/upload（带 AI 自动分类）
- 显示上传进度和结果

**改完后效果：**
- 点 Choose File → 选文件 → "上传中，AI 识别分类..." → "✅ 导入成功 · 模块：maintenance"

---

## 点6：知识学习与分析

**现状：**
- 粘贴文本 → AI 自动分类模块 → 写入数据库
- ❌ 没有对内容做摘要提取
- ❌ 没有提取关键词用于搜索增强
- ❌ 上传内容没有生成"知识卡片"

**需要改为：**
- 粘贴/上传后：
  1. AI 分析内容提取 3-5 个关键词
  2. 生成一句话摘要
  3. 自动归类到模块
  4. 存入数据库（content + keywords + summary）
- 搜索时用关键词匹配，提高命中率

**改完后效果：**
- 粘贴一段设备维修文档 → AI：关键词"封口机/加热条/维护" → 分类 maintenance
- 搜索"封口机维护" → 精准命中
