# CHUCHUTEA AI 知识库 - 系统审计报告

## 架构概览
```
用户浏览器 → Vercel(前端) → ngrok → 后端FastAPI → PostgreSQL + 豆包API
```
- 前端：React+TS 单页，部署在Vercel
- 后端：Python FastAPI，本地运行，通过ngrok暴露
- 数据库：PostgreSQL+pgvector，Docker运行
- 大模型：豆包 doubao-seed-1-6-251015

## 发现的6个问题

### 1. 俄语关键词搜索失败
- **原因**：中文用二字滑动窗口分词，俄语没有适配
- **影响**：俄语知识库命中率接近0%

### 2. 速度慢（首次显字延迟5-10秒）
- **原因**：流式输出先完整搜索知识库，再调LLM，串行
- **影响**：用户感知等待时间太长

### 3. 无对话上下文压缩
- **原因**：没有限制上下文长度
- **影响**：长对话越聊越慢，token消耗大

### 4. 双模型未实现
- **原因**：代码写死了豆包
- **影响**：普通文本也用vision模型，浪费

### 5. 左侧新对话按钮偶发失效
- **原因**：createConversation可能没正确触发

### 6. 知识库无品牌数据
- **影响**：问品牌问题都显示"仅供参考"

## 修复方案

| 问题 | 方案 | 预估 |
|------|------|------|
| 俄语搜索 | 西里尔字母按词分割 | 1行代码 |
| 速度 | 并行搜索+缓存 | 5行代码 |
| 上下文压缩 | 超过10条自动取最近5轮 | 3行代码 |
| 双模型 | 文字→DeepSeek，图片→豆包 | 10行代码 |
| 新对话 | 修复异常处理 | 5行代码 |
| 品牌数据 | 内置品牌知识种子条目 | API调用 |
